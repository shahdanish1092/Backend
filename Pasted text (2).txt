1. Architecture Overview
The target architecture has four clean layers with a single direction of control: the frontend never touches n8n directly, and n8n never writes to your app database directly. The diagram above shows the final state.
Responsibilities by layer:
The React frontend owns only presentation and user intent. It sends natural-language requests and structured form inputs to the FastAPI backend over REST. It never knows n8n's URL or credentials. The existing code is already well-structured for this — the api/ folder is clean, Shadcn UI is solid, and the page router is correctly protected. This layer needs extension, not replacement.
The FastAPI backend is the intelligence and orchestration layer. It parses user intent (via Groq/Llama), maps that intent to a workflow ID, calls n8n via its REST API to trigger execution, stores metadata in Postgres, and receives async callbacks from n8n when workflows complete. It owns user sessions, credential references, and audit logs. Your current backend has most of this scaffolding but has critical gaps (see Section 2).
n8n is the pure execution engine. It owns OAuth credentials for external services (Gmail, Google Sheets, etc.), runs workflows, and posts a webhook back to FastAPI when done. It should never be called directly by the frontend. It currently lives in a separate project — this is your most pressing infrastructure problem.
PostgreSQL on Railway is the source of truth for all application state. n8n uses its own internal SQLite/Postgres for workflow definitions; that stays separate. Your app DB stores users, workflow metadata, execution records, and credential references (not the actual OAuth tokens — those stay in n8n's credential vault).

3. n8n Integration Design
Use your existing n8n instance. Do not spin up a new one. The friction of managing two n8n instances far outweighs any isolation benefit at this stage. What you need is a network bridge between your Railway-hosted FastAPI and your Docker n8n.
How the backend triggers n8n:
Use n8n's REST API, not webhooks, for triggering. The flow is: frontend sends a request → FastAPI validates it, creates an execution record in Postgres with status pending, then calls POST https://<n8n-host>/api/v1/workflows/<workflow_id>/run with a Bearer token (your n8n API key). n8n starts the workflow asynchronously and returns a 200 immediately.
python# orchestration.py — trigger pattern
async def trigger_n8n_workflow(workflow_id: str, payload: dict, execution_id: str):
    headers = {"X-N8N-API-KEY": settings.N8N_API_KEY}
    body = {
        "workflowData": payload,
        "callbackUrl": f"{settings.BACKEND_URL}/api/webhooks/n8n/callback/{execution_id}"
    }
    async with httpx.AsyncClient() as client:
        resp = await client.post(
            f"{settings.N8N_BASE_URL}/api/v1/workflows/{workflow_id}/run",
            json=body, headers=headers, timeout=10.0
        )
    return resp.json()
How n8n posts results back:
Add an HTTP Request node at the end of every n8n workflow that POSTs to {BACKEND_URL}/api/webhooks/n8n/callback/{execution_id}. This endpoint updates the execution record status to completed or failed and stores the result payload. The frontend polls GET /api/executions/{execution_id} every 3 seconds until it gets a terminal status.
How workflows are created and managed:
For your current scope, pre-build workflows in n8n and store their IDs in your database. Do not attempt to generate workflows dynamically via the API — this is extremely complex and unnecessary for a capstone. The user's natural language request maps to a workflow ID via your intent router (Groq/Llama), and FastAPI triggers the pre-existing workflow with the user's parameters as the input payload.
Workflow discovery: Use GET /api/v1/workflows on n8n to fetch active workflow names and IDs at startup, cache them, and update your workflows table. This gives your sidebar real data.

    """Trigger an n8n workflow via REST API and register a callback URL."""

async def update_execution_status(
    db,
    execution_id: str,
    status: str,
    result: dict = None,
    error: str = None
):
    """Update execution_logs row status. 
    status must be one of: pending, running, completed, failed, timeout"""
    import json
    from datetime import datetime, timezone
    
    valid_statuses = {"pending", "running", "completed", "failed", "timeout"}
    if status not in valid_statuses:
        raise ValueError(f"Invalid status: {status}. Must be one of {valid_statuses}")
    
    update_fields = {
        "status": status,
        "updated_at": datetime.now(timezone.utc)
    }
    if result is not None:
        update_fields["result_payload"] = json.dumps(result)
    if error is not None:
        update_fields["error_message"] = error
    if status in ("completed", "failed", "timeout"):
        update_fields["completed_at"] = datetime.now(timezone.utc)
    
    await db.execute(
        """UPDATE execution_logs 
           SET status = :status,
               updated_at = :updated_at
               {result_clause}
               {error_clause}
               {completed_clause}
           WHERE id = :execution_id""".format(
            result_clause=", result_payload = :result_payload" if result is not None else "",
            error_clause=", error_message = :error_message" if error is not None else "",
            completed_clause=", completed_at = :completed_at" if status in ("completed", "failed", "timeout") else ""
        ),
        {"execution_id": execution_id, **update_fields}
    )
    await db.commit()

import httpx
import os
import base64

N8N_BASE_URL = os.getenv("N8N_BASE_URL", "")
N8N_API_KEY = os.getenv("N8N_API_KEY", "")
N8N_BASIC_AUTH_USER = os.getenv("N8N_BASIC_AUTH_USER", "")
N8N_BASIC_AUTH_PASSWORD = os.getenv("N8N_BASIC_AUTH_PASSWORD", "")
BACKEND_PUBLIC_URL = os.getenv("BACKEND_PUBLIC_URL", "")
N8N_CALLBACK_SECRET = os.getenv("N8N_CALLBACK_SECRET", "")

def _n8n_headers() -> dict:
    """Build headers required for every n8n API call."""
    basic = base64.b64encode(
        f"{N8N_BASIC_AUTH_USER}:{N8N_BASIC_AUTH_PASSWORD}".encode()
    ).decode()
    return {
        "X-N8N-API-KEY": N8N_API_KEY,
        "Authorization": f"Basic {basic}",
        "Content-Type": "application/json",
        "ngrok-skip-browser-warning": "true",
    }

async def trigger_n8n_workflow(workflow_id: str, payload: dict, execution_id: str) -> dict:
    """Trigger an n8n workflow and pass a callback URL for async result reporting."""
    callback_url = f"{BACKEND_PUBLIC_URL}/api/webhooks/n8n/callback/{execution_id}"
    body = {
        "workflowData": {
            **payload,
            "_callback_url": callback_url,
            "_execution_id": execution_id,
            "_secret": N8N_CALLBACK_SECRET,
        }
    }
    async with httpx.AsyncClient(timeout=15.0) as client:
        resp = await client.post(
            f"{N8N_BASE_URL}/api/v1/workflows/{workflow_id}/run",
            json=body,
            headers=_n8n_headers(),
        )
        resp.raise_for_status()
    return resp.json()

async def get_n8n_workflows() -> list:
    """Fetch all workflows from n8n for the sync endpoint."""
    async with httpx.AsyncClient(timeout=10.0) as client:
        resp = await client.get(
            f"{N8N_BASE_URL}/api/v1/workflows",
            headers=_n8n_headers(),
        )
        resp.raise_for_status()
    return resp.json().get("data", [])

async def update_execution_status(
    db,
    execution_id: str,
    status: str,
    result: dict = None,
    error: str = None,
):
    """Update an execution_logs row. status: pending|running|completed|failed|timeout"""
    now = datetime.now(timezone.utc)
    params = {"id": execution_id, "status": status, "updated_at": now}
    set_clauses = ["status = :status", "updated_at = :updated_at"]

    if result is not None:
        params["result_payload"] = json.dumps(result)
        set_clauses.append("result_payload = :result_payload")
    if error is not None:
        params["error_message"] = error
        set_clauses.append("error_message = :error_message")
    if status in ("completed", "failed", "timeout"):
        params["completed_at"] = now
        set_clauses.append("completed_at = :completed_at")

    sql = f"UPDATE execution_logs SET {', '.join(set_clauses)} WHERE id = :id"
    await db.execute(sql, params)
    await db.commit()

async def log_execution(db, user_email: str, workflow_id: str, input_payload: dict) -> str:
    """Create a new execution_logs row and return the execution_id."""
    execution_id = str(uuid4())
    now = datetime.now(timezone.utc)
    await db.execute(
        """INSERT INTO execution_logs 
           (id, user_email, workflow_id, status, input_payload, started_at, updated_at)
           VALUES (:id, :user_email, :workflow_id, 'pending', :input_payload, :started_at, :updated_at)""",
        {
            "id": execution_id,
            "user_email": user_email,
            "workflow_id": workflow_id,
            "input_payload": json.dumps(input_payload),
            "started_at": now,
            "updated_at": now,
        }
    )
    await db.commit()
    return execution_id

export const chatApi = {
  sendMessage: async (text, activeConnectors, userEmail) => {
    const response = await apiClient.post('/chat/message', {
      text,
      active_connectors: activeConnectors,
      user_email: userEmail,
    });
    return response.data;
    // NO catch/mock — let errors surface
  },
  getExecution: async (executionId) => {
    const response = await apiClient.get(`/executions/${executionId}`);
    return response.data;
  },
  getActiveExecutions: async () => {
    const response = await apiClient.get('/executions?status=running&limit=5');
    return response.data;
  },
};
